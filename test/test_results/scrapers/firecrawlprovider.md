# Graphics processing unit

Graphics processing unit

Specialized electronic circuit that accelerates graphics

"GPU" redirects here. For other uses, see [GPU (disambiguation)](https://en.wikipedia.org/wiki/GPU_(disambiguation) "GPU (disambiguation)").

For an expansion card that contains a graphics processing unit, see [Graphics card](https://en.wikipedia.org/wiki/Graphics_card "Graphics card").

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/c/cd/Generic_block_diagram_of_a_GPU.svg/960px-Generic_block_diagram_of_a_GPU.svg.png)](https://en.wikipedia.org/wiki/File:Generic_block_diagram_of_a_GPU.svg) The components of a GPU.

A **graphics processing unit** ( **GPU**) is a specialized [electronic circuit](https://en.wikipedia.org/wiki/Electronic_circuit "Electronic circuit") designed for [digital image processing](https://en.wikipedia.org/wiki/Digital_image_processing "Digital image processing") and to accelerate [computer graphics](https://en.wikipedia.org/wiki/Computer_graphics "Computer graphics"), being present either as a component on a discrete [graphics card](https://en.wikipedia.org/wiki/Graphics_card "Graphics card") or embedded on [motherboards](https://en.wikipedia.org/wiki/Motherboard "Motherboard"), [mobile phones](https://en.wikipedia.org/wiki/Mobile_phone "Mobile phone"), [personal computers](https://en.wikipedia.org/wiki/Personal_computer "Personal computer"), [workstations](https://en.wikipedia.org/wiki/Workstation "Workstation"), and [game consoles](https://en.wikipedia.org/wiki/Video_game_console "Video game console"). GPUs are also increasingly being used for [artificial intelligence](https://en.wikipedia.org/wiki/Artificial_intelligence "Artificial intelligence") (AI) processing due to linear algebra acceleration, which is also used extensively in graphics processing.

Although there is no single definition of the term, and it may be used to describe any video display system, in modern use a GPU includes the ability to internally perform the calculations needed for various graphics tasks, like rotating and scaling 3D images, and often the additional ability to run custom programs known as [shaders](https://en.wikipedia.org/wiki/Shader "Shader"). This contrasts with earlier graphics controllers known as [video display controllers](https://en.wikipedia.org/wiki/Video_display_controller "Video display controller") which had no internal calculation capabilities, or [blitters](https://en.wikipedia.org/wiki/Blitter "Blitter"), which performed only basic memory movement operations. The modern GPU emerged during the 1990s, adding the ability to perform operations like drawing lines and text without [CPU](https://en.wikipedia.org/wiki/Central_processing_unit "Central processing unit") help, and later adding 3D functionality.

Graphics functions are generally independent and this lends these tasks to being implemented on separate calculation engines. Modern GPUs include hundreds, or thousands, of calculation units. This made them useful for non-graphic calculations involving [embarrassingly parallel](https://en.wikipedia.org/wiki/Embarrassingly_parallel "Embarrassingly parallel") problems due to their [parallel structure](https://en.wikipedia.org/wiki/Parallel_computing "Parallel computing"). The ability of GPUs to rapidly perform vast numbers of calculations has led to their adoption in diverse fields including [artificial intelligence](https://en.wikipedia.org/wiki/Artificial_intelligence "Artificial intelligence") (AI) where they excel at handling data-intensive and computationally demanding tasks. Other non-graphical uses include the training of [neural networks](https://en.wikipedia.org/wiki/Artificial_neural_network#Training "Artificial neural network") and [cryptocurrency mining](https://en.wikipedia.org/wiki/GPU_mining "GPU mining").

## Computational functions

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/8/88/AMD_HD5470_GPU.JPG/500px-AMD_HD5470_GPU.JPG)](https://en.wikipedia.org/wiki/File:AMD_HD5470_GPU.JPG) The ATI HD5470 GPU (above, with copper [heatpipe](https://en.wikipedia.org/wiki/Heatpipe "Heatpipe") attached) features [UVD](https://en.wikipedia.org/wiki/UVD "UVD") 2.1 which enables it to decode AVC and VC-1 video formats.

Several factors of GPU construction affect the performance of the card for real-time rendering, such as the size of the connector pathways in the [semiconductor device fabrication](https://en.wikipedia.org/wiki/Semiconductor_device_fabrication "Semiconductor device fabrication"), the [clock signal](https://en.wikipedia.org/wiki/Clock_signal "Clock signal") frequency, and the number and size of various on-chip memory [caches](https://en.wikipedia.org/wiki/CPU_cache "CPU cache"). Performance is also affected by the number of streaming multiprocessors (SM) for NVidia GPUs, or compute units (CU) for AMD GPUs, or Xe cores for Intel Xe-based GPUs, which describe the number of on-silicon processor core units within the GPU chip that perform the core calculations, typically working in parallel with other SM/CUs on the GPU. GPU performance is typically measured in floating point operations per second ( [FLOPS](https://en.wikipedia.org/wiki/FLOPS "FLOPS")); Modern GPUs typically deliver performance measured in teraflops (TFLOPS). This is an estimated performance measure, and should not be treated as fact, as other factors can affect actual performance.[\[3\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-:3-3)

Modern GPUs also include dedicated hardware blocks for [ray tracing](https://en.wikipedia.org/wiki/Ray_tracing_(graphics) "Ray tracing (graphics)"), [video encoding](https://en.wikipedia.org/wiki/Video_Acceleration_API "Video Acceleration API"), and [AI acceleration](https://en.wikipedia.org/wiki/AI_accelerator "AI accelerator").

## GPU forms

In personal computers, there are two main forms of GPUs: dedicated graphics (also called discrete graphics) and integrated graphics (also called shared graphics solutions, integrated graphics processors (IGP), or unified memory architecture (UMA). [\[4\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-:4-4)

### Dedicated graphics processing unit

See also: [Video card](https://en.wikipedia.org/wiki/Video_card "Video card")

_Dedicated graphics processing units_ use on board [RAM](https://en.wikipedia.org/wiki/Random-access_memory "Random-access memory") that is dedicated to the GPU rather than relying on the computer's main system memory. This RAM is usually specially selected for the expected serial workload of the graphics card, such as [GDDR SDRAM](https://en.wikipedia.org/wiki/GDDR_SDRAM "GDDR SDRAM"). This has massive performance benefits, but the caveat of "choking" when running out of dedicated memory, worsening performance.

Technologies such as [Scalable Link Interface](https://en.wikipedia.org/wiki/Scalable_Link_Interface "Scalable Link Interface") (SLI), [NVLink](https://en.wikipedia.org/wiki/NVLink "NVLink"), and [CrossFire](https://en.wikipedia.org/wiki/ATI_CrossFire "ATI CrossFire") allow multiple GPUs to draw images simultaneously for a single screen, increasing the processing power available for graphics. These technologies, however, are increasingly uncommon; most games do not fully use multiple GPUs, as most users cannot afford them.[\[5\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-CSL_1-5)[\[6\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-6)[\[7\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-7)\[ _[bettersourceneeded](https://en.wikipedia.org/wiki/Wikipedia:Verifiability#Questionable_sources "Wikipedia:Verifiability")_\] Multiple GPUs are still used on supercomputers (such as in [Summit](https://en.wikipedia.org/wiki/Summit_(supercomputer) "Summit (supercomputer)")); on workstations to accelerate video (processing multiple videos at once)[\[8\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-8)[\[4\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-:4-4)[\[9\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-9) and 3D rendering;[\[10\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-10) for [visual effects](https://en.wikipedia.org/wiki/Visual_effects "Visual effects") (VFX);[\[11\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-11) [general purpose graphics processing unit](https://en.wikipedia.org/wiki/General-purpose_computing_on_graphics_processing_units "General-purpose computing on graphics processing units") (GPGPU) workloads and for simulations,[\[12\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-12) and in AI to expedite training, as is the case with Nvidia's lineup of DGX workstations and servers.\[ _[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_\]

### Integrated graphics processing unit

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/b/bd/Motherboard_diagram.svg/500px-Motherboard_diagram.svg.png)](https://en.wikipedia.org/wiki/File:Motherboard_diagram.svg) The position of an integrated GPU in a [northbridge](https://en.wikipedia.org/wiki/Northbridge_(computing) "Northbridge (computing)")/ [southbridge](https://en.wikipedia.org/wiki/Southbridge_(computing) "Southbridge (computing)") system layout.[![](https://upload.wikimedia.org/wikipedia/commons/thumb/0/0c/A790GXH-128M-Motherboard.jpg/500px-A790GXH-128M-Motherboard.jpg)](https://en.wikipedia.org/wiki/File:A790GXH-128M-Motherboard.jpg) An [ASRock](https://en.wikipedia.org/wiki/ASRock "ASRock") motherboard with integrated graphics, which has HDMI, VGA and DVI-out ports.

_Integrated graphics processing units_ (IGPU), also called _integrated graphics_, _shared graphics solutions_, _integrated graphics processors_ (IGP), or _unified memory architectures_ (UMA) use a portion of a computer's system RAM rather than dedicated graphics memory. IGPs can be integrated onto a motherboard as part of its [northbridge](https://en.wikipedia.org/wiki/Northbridge_(computing) "Northbridge (computing)") chipset,[\[13\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-13) or on the same [die (integrated circuit)](https://en.wikipedia.org/wiki/Die_(integrated_circuit) "Die (integrated circuit)") with the CPU, such as [Accelerated Processing Unit](https://en.wikipedia.org/wiki/AMD_APU "AMD APU") (AMD APU) or [Intel HD Graphics](https://en.wikipedia.org/wiki/Intel_HD_Graphics "Intel HD Graphics"). IGPUs and APUs are less costly to implement than dedicated graphics processing, but tend to be less capable. Integrated graphics processing was considered unfit for 3D games or graphically intensive programs but could run less intensive programs such as Adobe Flash. Examples of such IGPs would be offerings from SiS and VIA circa 2004.[\[14\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-14) However, modern integrated graphics processors such as [AMD Accelerated Processing Units](https://en.wikipedia.org/wiki/AMD_Accelerated_Processing_Unit "AMD Accelerated Processing Unit") and [Intel Graphics Technology](https://en.wikipedia.org/wiki/Intel_Graphics_Technology "Intel Graphics Technology") can even handle Triple A games at lower settings.\[ _[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_\]

Because GPU computations are memory-intensive, integrated processing may compete with the CPU for relatively slow system RAM, as it has minimal or no dedicated video memory. IGPUs use system memory with bandwidth up to a current maximum of 128 gigabytes per second, whereas a discrete graphics card may have a bandwidth[\[15\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-15) of more than 1000 gigabytes per second between its [video random access memory](https://en.wikipedia.org/wiki/Video_random_access_memory "Video random access memory") (VRAM) and GPU core. This [memory bus](https://en.wikipedia.org/wiki/Memory_bus "Memory bus") bandwidth can limit the performance of the IGPU, though [multi-channel memory](https://en.wikipedia.org/wiki/Multi-channel_memory_architecture "Multi-channel memory architecture") can mitigate this deficiency.[\[16\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Coelho-16)

On systems with "Unified Memory Architecture" (UMA), including modern AMD processors with integrated graphics,[\[17\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-17) modern Intel processors with integrated graphics,[\[18\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-18) Apple processors, and modern consoles, the CPU cores and the GPU block share the same pool of RAM and memory address space.\[ _[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_\]

### Stream processing and general purpose GPUs (GPGPU)

Main articles: [GPGPU](https://en.wikipedia.org/wiki/GPGPU "GPGPU") and [Stream processing](https://en.wikipedia.org/wiki/Stream_processing "Stream processing")

It is common to use a general purpose graphics processing unit (GPGPU) as a modified form of stream processor or a [vector processor](https://en.wikipedia.org/wiki/Vector_processor "Vector processor"), running [compute kernels](https://en.wikipedia.org/wiki/Compute_kernel "Compute kernel"). This turns the massive computational power of a modern graphics accelerator's shader pipeline into general-purpose computing power. In certain applications requiring massive vector operations, this can yield several orders of magnitude higher performance than a conventional CPU. The two largest discrete GPU designers, [AMD](https://en.wikipedia.org/wiki/AMD "AMD") and [Nvidia](https://en.wikipedia.org/wiki/Nvidia "Nvidia"), are pursuing this approach with an array of applications.\[ _[as of?](https://en.wikipedia.org/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items "Wikipedia:Manual of Style/Dates and numbers")_\] Nvidia and AMD collaborated with [Stanford University](https://en.wikipedia.org/wiki/Stanford_University "Stanford University") to create a GPU-based client for the [Folding@home](https://en.wikipedia.org/wiki/Folding@home "Folding@home") distributed computing project for protein folding calculations. In certain circumstances, the GPU calculates forty times faster than the CPUs traditionally used by such applications.[\[19\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-19)[\[20\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-20)

GPU-based high performance computers play a significant role in large-scale modelling. Three of the ten most powerful supercomputers in the world take advantage of GPU acceleration.[\[21\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-21)\[ _[needs update](https://en.wikipedia.org/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items "Wikipedia:Manual of Style/Dates and numbers")_\]

Since 2005 there has been interest in using the performance offered by GPUs for [evolutionary computation](https://en.wikipedia.org/wiki/Evolutionary_computation "Evolutionary computation") in general, and for accelerating the [fitness](https://en.wikipedia.org/wiki/Fitness_(genetic_algorithm) "Fitness (genetic algorithm)") evaluation in [genetic programming](https://en.wikipedia.org/wiki/Genetic_programming "Genetic programming") in particular.\[ _[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_\] Most approaches compile [linear](https://en.wikipedia.org/wiki/Linear_genetic_programming "Linear genetic programming") or [tree programs](https://en.wikipedia.org/wiki/Genetic_programming "Genetic programming") on the host PC and transfer the executable to the GPU to be run. Typically a performance advantage is only obtained by running the single active program simultaneously on many example problems in parallel, using the GPU's [single instruction, multiple data](https://en.wikipedia.org/wiki/Single_instruction,_multiple_data "Single instruction, multiple data") (SIMD) architecture.[\[22\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-22)\[ _[needs update](https://en.wikipedia.org/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items "Wikipedia:Manual of Style/Dates and numbers")_\] Substantial acceleration can also be obtained by not compiling the programs, and instead transferring them to the GPU, to be interpreted there.[\[23\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-23)

### External GPU (eGPU)

A GPU can be attached to some external bus of a notebook. [PCI Express](https://en.wikipedia.org/wiki/PCI_Express "PCI Express") is the only bus used for this purpose.\[ _[as of?](https://en.wikipedia.org/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items "Wikipedia:Manual of Style/Dates and numbers")_\] The port may be, for example, an [ExpressCard](https://en.wikipedia.org/wiki/ExpressCard "ExpressCard") or [mPCIe](https://en.wikipedia.org/wiki/PCI_Express#PCI_Express_Mini_Card "PCI Express") port (PCIe ×1, up to 5 or 2.5 gigabits per second respectively), a [Thunderbolt](https://en.wikipedia.org/wiki/Thunderbolt_(interface) "Thunderbolt (interface)") 1, 2, or 3 port (PCIe ×4, up to 10, 20, or 40 gigabits per second respectively), a [USB4 port with Thunderbolt compatibility](https://en.wikipedia.org/wiki/Thunderbolt_(interface)#USB4 "Thunderbolt (interface)"), or an [OCuLink](https://en.wikipedia.org/wiki/OCuLink "OCuLink") port. Those ports are only available on certain notebook systems.[\[24\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-24) eGPU enclosures include their own power supply (PSU), because powerful GPUs can consume hundreds of watts.[\[25\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-25)

### 1960s

[![](https://upload.wikimedia.org/wikipedia/en/d/d6/Adage_Graphics_Terminal_brochure_-_crop.jpg)](https://en.wikipedia.org/wiki/File:Adage_Graphics_Terminal_brochure_-_crop.jpg) Adage Graphics terminal from 1968 brochure

Dedicated 3D graphics hardware dates back to graphic [terminals](https://en.wikipedia.org/wiki/Computer_terminal "Computer terminal") such as the [Adage](https://en.wikipedia.org/wiki/Adage,_Inc. "Adage, Inc.") AGT-30 from 1967 with [analog](https://en.wikipedia.org/wiki/Analog_electronics "Analog electronics") [matrix](https://en.wikipedia.org/wiki/Matrix_(mathematics) "Matrix (mathematics)") processors. In 1969 [Evans & Sutherland](https://en.wikipedia.org/wiki/Evans_&_Sutherland "Evans & Sutherland") (E&S) introduced the [Line Drawing System-1](https://en.wikipedia.org/wiki/Line_Drawing_System-1 "Line Drawing System-1") (LDS-1), which was the first all-digital system to provide matrix multiplication. Also in 1969, the low-cost graphics terminal [IMLAC PDS-1](https://en.wikipedia.org/wiki/IMLAC "IMLAC") was introduced. It later saw use as an early 3D gaming machine with the likes of Maze War.

### 1970s

In the 1970s, the term "GPU" originally stood for graphics processor unit and described a programmable processing unit working independently from the CPU that was responsible for graphics manipulation and output.[\[1\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-:1-1)[\[2\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-:2-2)

In professional hardware, in 1972 [PLATO IV](https://en.wikipedia.org/wiki/PLATO_(computer_system) "PLATO (computer system)") system becomes operational at the [University of Illinois Urbana-Champaign](https://en.wikipedia.org/wiki/University_of_Illinois_Urbana-Champaign "University of Illinois Urbana-Champaign"). Between around 1973 and 1978, several networked multiplayer wireframe 3D games are implemented and popularized by users of the system. Also in 1972, the E&S Continuous Tone 1 (CT1) "Watkins box" system (consisting of an E&S LDS-2 and [Shaded Picture System](https://en.wikipedia.org/wiki/Shaded_Picture_System "Shaded Picture System")) is delivered to [Case Western Reserve University](https://en.wikipedia.org/wiki/Case_Western_Reserve_University "Case Western Reserve University"). It offered the first real-time [Gouraud shading](https://en.wikipedia.org/wiki/Gouraud_shading "Gouraud shading"). In 1975, a joint effort between Evans & Sutherland Computer Corporation and the University of Utah's computer graphics department results in the first ever MOSFET video [framebuffer](https://en.wikipedia.org/wiki/Framebuffer "Framebuffer"), capable of color and smooth shading. E&S Continuous Tone 3 (CT3) system was delivered in 1977 to [Lufthansa](https://en.wikipedia.org/wiki/Lufthansa "Lufthansa") for pilot training using computer simulation. It was the first graphics system capable of real-time texture mapping.\[ _[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_\] Ikonas made graphics systems with 8- and 24-bit graphics and 3D acceleration in the late 70s.[\[26\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Peddie-26)

[Arcade system boards](https://en.wikipedia.org/wiki/Arcade_system_board "Arcade system board") have used specialized 2D graphics circuits since the 1970s. In early video game hardware, [RAM](https://en.wikipedia.org/wiki/Random-access_memory "Random-access memory") for frame buffers was expensive, so video chips composited data together as the display was being scanned out on the monitor.[\[27\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-27)

A specialized [barrel shifter](https://en.wikipedia.org/wiki/Barrel_shifter "Barrel shifter") circuit helped the CPU animate the [framebuffer](https://en.wikipedia.org/wiki/Framebuffer "Framebuffer") graphics for various 1970s [arcade video games](https://en.wikipedia.org/wiki/Arcade_video_game "Arcade video game") from [Midway](https://en.wikipedia.org/wiki/Midway_Games "Midway Games") and [Taito](https://en.wikipedia.org/wiki/Taito "Taito"), such as _[Gun Fight](https://en.wikipedia.org/wiki/Gun_Fight "Gun Fight")_ (1975), _[Sea Wolf](https://en.wikipedia.org/wiki/Sea_Wolf_(video_game) "Sea Wolf (video game)")_ (1976), and _[Space Invaders](https://en.wikipedia.org/wiki/Space_Invaders "Space Invaders")_ (1978).[\[28\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-28) The [Namco Galaxian](https://en.wikipedia.org/wiki/Namco_Galaxian "Namco Galaxian") arcade system in 1979 used specialized [graphics hardware](https://en.wikipedia.org/wiki/Graphics_hardware "Graphics hardware") that supported [RGB color](https://en.wikipedia.org/wiki/RGB_color_model "RGB color model"), multi-colored sprites, and [tilemap](https://en.wikipedia.org/wiki/Tile_engine "Tile engine") backgrounds.[\[29\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-29) The Galaxian hardware was widely used during the [golden age of arcade video games](https://en.wikipedia.org/wiki/Golden_age_of_arcade_video_games "Golden age of arcade video games"), by game companies such as [Irem](https://en.wikipedia.org/wiki/Irem "Irem"), [Konami](https://en.wikipedia.org/wiki/Konami "Konami"), Midway, [Nichibutsu](https://en.wikipedia.org/wiki/Nichibutsu "Nichibutsu"), [Sega](https://en.wikipedia.org/wiki/Sega "Sega"), and [Taito](https://en.wikipedia.org/wiki/Taito "Taito").[\[30\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-30)

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/4/43/ANTIC_chip_on_an_Atari_130XE_motherboard.jpg/500px-ANTIC_chip_on_an_Atari_130XE_motherboard.jpg)](https://en.wikipedia.org/wiki/File:ANTIC_chip_on_an_Atari_130XE_motherboard.jpg) Atari [ANTIC](https://en.wikipedia.org/wiki/ANTIC "ANTIC") microprocessor on an Atari 130XE motherboard

The [Atari 2600](https://en.wikipedia.org/wiki/Atari_2600 "Atari 2600") in 1977 used a video shifter called the [Television Interface Adaptor](https://en.wikipedia.org/wiki/Television_Interface_Adaptor "Television Interface Adaptor").[\[31\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-31) [Atari 8-bit computers](https://en.wikipedia.org/wiki/Atari_8-bit_computers "Atari 8-bit computers") (1979) had [ANTIC](https://en.wikipedia.org/wiki/ANTIC "ANTIC"), a video processor which interpreted instructions describing a " [display list](https://en.wikipedia.org/wiki/Display_list "Display list")"—the way the scan lines map to specific [bitmapped](https://en.wikipedia.org/wiki/Bitmapped "Bitmapped") or character modes and where the memory is stored (so there did not need to be a contiguous frame buffer).\[ _[clarification needed](https://en.wikipedia.org/wiki/Wikipedia:Please_clarify "Wikipedia:Please clarify")_\][\[32\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-32) [6502](https://en.wikipedia.org/wiki/6502 "6502") [machine code](https://en.wikipedia.org/wiki/Machine_code "Machine code") [subroutines](https://en.wikipedia.org/wiki/Subroutine "Subroutine") could be triggered on [scan lines](https://en.wikipedia.org/wiki/Scan_line "Scan line") by setting a bit on a display list instruction.\[ _[clarification needed](https://en.wikipedia.org/wiki/Wikipedia:Please_clarify "Wikipedia:Please clarify")_\][\[33\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-33) ANTIC also supported smooth [vertical](https://en.wikipedia.org/wiki/Vertical_scrolling "Vertical scrolling") and [horizontal scrolling](https://en.wikipedia.org/wiki/Horizontal_scrolling "Horizontal scrolling") independent of the CPU.[\[34\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-34)

### 1980s

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/6/69/SGI_GE25A_Geometry_Engine_die.jpg/500px-SGI_GE25A_Geometry_Engine_die.jpg)](https://en.wikipedia.org/wiki/File:SGI_GE25A_Geometry_Engine_die.jpg) Geometry Engine integrated circuit

In the 1980s significant advancements were made in professional 3D graphics hardware. Perhaps most impactful was the 1981 development of the [Geometry Engine](https://en.wikipedia.org/wiki/Geometry_Engine "Geometry Engine"), a [VLSI](https://en.wikipedia.org/wiki/VLSI "VLSI") [vector](https://en.wikipedia.org/wiki/Vector_(mathematics_and_physics) "Vector (mathematics and physics)") processor [ASIC](https://en.wikipedia.org/wiki/ASIC "ASIC") designed by [Jim Clark](https://en.wikipedia.org/wiki/James_H._Clark "James H. Clark") and [Marc Hannah](https://en.wikipedia.org/wiki/Marc_Hannah "Marc Hannah") at [Stanford University](https://en.wikipedia.org/wiki/Stanford_University "Stanford University"). This processor is the forerunner of modern [tensor](https://en.wikipedia.org/wiki/Tensor_(mathematics) "Tensor (mathematics)") cores and other similar processors marketed for graphics and AI. The Geometry Engine went on to be used in [Silicon Graphics](https://en.wikipedia.org/wiki/Silicon_Graphics_Incorporated "Silicon Graphics Incorporated") [workstations](https://en.wikipedia.org/wiki/Workstations "Workstations") for many years. Silicon Graphics's first product, shipped in November 1983, was the IRIS 1000, a terminal with hardware-accelerated 3D graphics based on the Geometry Engine.[\[26\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Peddie-26) The Geometry Engine was capable of approximately 6 million operations per second.[\[35\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-35)

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/2/28/NEC_D7220.jpg/500px-NEC_D7220.jpg)](https://en.wikipedia.org/wiki/File:NEC_D7220.jpg) NEC [μPD7220](https://en.wikipedia.org/wiki/%CE%9CPD7220 "ΜPD7220") A

The 1981 [NEC μPD7220](https://en.wikipedia.org/wiki/NEC_%CE%BCPD7220 "NEC μPD7220") was the first implementation of a [personal computer](https://en.wikipedia.org/wiki/Personal_computer "Personal computer") graphics display processor as a single [large-scale integration](https://en.wikipedia.org/wiki/Large-scale_integration "Large-scale integration") (LSI) [integrated circuit](https://en.wikipedia.org/wiki/Integrated_circuit "Integrated circuit") chip. This enabled the design of low-cost, high-performance video graphics cards such as those from [Number Nine Visual Technology](https://en.wikipedia.org/wiki/Number_Nine_Visual_Technology "Number Nine Visual Technology"). It became the best-known GPU until the mid-1980s.[\[36\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-36) It was the first fully integrated [VLSI](https://en.wikipedia.org/wiki/VLSI "VLSI") (very large-scale integration) [metal–oxide–semiconductor](https://en.wikipedia.org/wiki/Metal%E2%80%93oxide%E2%80%93semiconductor "Metal–oxide–semiconductor") ( [NMOS](https://en.wikipedia.org/wiki/NMOS_logic "NMOS logic")) graphics display processor for PCs, supported up to [1024×1024 resolution](https://en.wikipedia.org/wiki/XGA "XGA"), and laid the foundations for the PC graphics market. It was used in a number of graphics cards and was licensed for clones such as the Intel 82720, the first of [Intel's graphics processing units](https://en.wikipedia.org/wiki/List_of_Intel_graphics_processing_units "List of Intel graphics processing units").[\[37\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-37) The [Williams Electronics](https://en.wikipedia.org/wiki/WMS_Industries "WMS Industries") arcade games _[Robotron: 2084](https://en.wikipedia.org/wiki/Robotron:_2084 "Robotron: 2084")_, _[Joust](https://en.wikipedia.org/wiki/Joust_(video_game) "Joust (video game)")_, _[Sinistar](https://en.wikipedia.org/wiki/Sinistar "Sinistar")_, and _[Bubbles](https://en.wikipedia.org/wiki/Bubbles_(video_game) "Bubbles (video game)")_, all released in 1982, contain custom [blitter](https://en.wikipedia.org/wiki/Blitter "Blitter") chips for operating on 16-color bitmaps.[\[38\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-38)[\[39\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-39)

In 1984, [Hitachi](https://en.wikipedia.org/wiki/Hitachi "Hitachi") released the ARTC HD63484, the first major [CMOS](https://en.wikipedia.org/wiki/CMOS "CMOS") graphics processor for personal computers. The ARTC could display up to [4K resolution](https://en.wikipedia.org/wiki/4K_resolution "4K resolution") when in [monochrome](https://en.wikipedia.org/wiki/Monochrome "Monochrome") mode. It was used in a number of graphics cards and terminals during the late 1980s.[\[40\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-40)

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/0/0e/Commodore_Amiga_1000_-_main_board_-_MOS_8367R0-7824.jpg/960px-Commodore_Amiga_1000_-_main_board_-_MOS_8367R0-7824.jpg)](https://en.wikipedia.org/wiki/File:Commodore_Amiga_1000_-_main_board_-_MOS_8367R0-7824.jpg) MOS 8367R0 – Agnus

In 1985, the [Amiga](https://en.wikipedia.org/wiki/Amiga "Amiga") was released with a custom graphics chip called [Agnus](https://en.wikipedia.org/wiki/MOS_Technology_Agnus "MOS Technology Agnus") including a blitter for bitmap manipulation, line drawing, and area fill. It also included a [coprocessor](https://en.wikipedia.org/wiki/Coprocessor "Coprocessor") with its own simple instruction set, that was capable of manipulating graphics hardware registers in sync with the video beam (e.g. for per-scanline palette switches, [sprite multiplexing](https://en.wikipedia.org/wiki/Sprite_multiplexing "Sprite multiplexing"), and hardware windowing), or driving the blitter.\[ _[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_\]

Also in 1985, IBM released the [Professional Graphics Controller](https://en.wikipedia.org/wiki/Professional_Graphics_Controller "Professional Graphics Controller"), designed by later to be Nvidia co-founder [Curtis Priem](https://en.wikipedia.org/wiki/Curtis_Priem "Curtis Priem"), which was a rudimentary 3D card with 640 × 480 256-color graphics which used a dedicated CPU to draw graphics independently of the main system. It was used as the basis of cards by a number of makers (including [Matrox](https://en.wikipedia.org/wiki/Matrox "Matrox")) and its analog RGB signaling led directly to the VGA video standard.[\[26\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Peddie-26) Priem later in the 80s worked on the influential [Sun Microsystems](https://en.wikipedia.org/wiki/Sun_Microsystems "Sun Microsystems") GX (also known as cgsix) accelerated 2D graphics card.[\[41\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Priem-41)

In 1986, [Texas Instruments](https://en.wikipedia.org/wiki/Texas_Instruments "Texas Instruments") released the [TMS34010](https://en.wikipedia.org/wiki/TMS34010 "TMS34010"), the first fully programmable graphics processor.[\[42\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-42) It could run general-purpose code but also had a graphics-oriented instruction set. During 1990–1992, this chip became the basis of the [Texas Instruments Graphics Architecture](https://en.wikipedia.org/wiki/Texas_Instruments_Graphics_Architecture "Texas Instruments Graphics Architecture") ("TIGA") [Windows accelerator](https://en.wikipedia.org/wiki/Windows_accelerator "Windows accelerator") cards.

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/2/22/IBM_8514.jpg/500px-IBM_8514.jpg)](https://en.wikipedia.org/wiki/File:IBM_8514.jpg) The [IBM 8514](https://en.wikipedia.org/wiki/IBM_8514 "IBM 8514") Micro Channel adapter, with memory add-on

Following in 1987, the [IBM 8514](https://en.wikipedia.org/wiki/IBM_8514 "IBM 8514") graphics system was released. It was one of the first video cards for [IBM PC compatibles](https://en.wikipedia.org/wiki/IBM_PC_compatible "IBM PC compatible") that implemented [fixed-function](https://en.wikipedia.org/wiki/Fixed-function "Fixed-function") 2D primitives in [electronic hardware](https://en.wikipedia.org/wiki/Electronic_hardware "Electronic hardware"). [Sharp](https://en.wikipedia.org/wiki/Sharp_Corporation "Sharp Corporation")'s [X68000](https://en.wikipedia.org/wiki/X68000 "X68000"), released in 1987, used a custom graphics chipset[\[43\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-43) with a 65,536 color palette and hardware support for sprites, scrolling, and multiple playfields.[\[44\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-44) It served as a development machine for [Capcom](https://en.wikipedia.org/wiki/Capcom "Capcom")'s [CP System](https://en.wikipedia.org/wiki/CP_System "CP System") arcade board. Fujitsu's [FM Towns](https://en.wikipedia.org/wiki/FM_Towns "FM Towns") computer, released in 1989, had support for a 16,777,216 color palette.[\[45\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-45)

For context, [IBM](https://en.wikipedia.org/wiki/IBM "IBM") also introduced its [Video Graphics Array](https://en.wikipedia.org/wiki/Video_Graphics_Array "Video Graphics Array") (VGA) display system in 1987, with a maximum resolution of 640 × 480 pixels. Unlike 8514/A, VGA had no hardware acceleration features. In November 1988, [NEC Home Electronics](https://en.wikipedia.org/wiki/NEC "NEC") announced its creation of the [Video Electronics Standards Association](https://en.wikipedia.org/wiki/Video_Electronics_Standards_Association "Video Electronics Standards Association") (VESA) to develop and promote a [Super VGA](https://en.wikipedia.org/wiki/Super_VGA "Super VGA") (SVGA) [computer display standard](https://en.wikipedia.org/wiki/Computer_display_standard "Computer display standard") as a successor to VGA. Super VGA enabled [graphics display resolutions](https://en.wikipedia.org/wiki/Graphics_display_resolution "Graphics display resolution") up to 800 × 600 [pixels](https://en.wikipedia.org/wiki/Pixel "Pixel"), a 56% increase.[\[46\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-InfoWorld_1988-11-14-46)

In 1988 SGI sold IRIS workstation graphics with 10-12 Geometry Engines and introduced the [IrisVision](https://en.wikipedia.org/wiki/IrisVision "IrisVision") add-in board for IBM MicroChannel bus ( [RS/6000](https://en.wikipedia.org/wiki/IBM_RS/6000 "IBM RS/6000")) based on the Geometry Engine as well.[\[26\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Peddie-26)

In 1988 as well, the first dedicated [polygonal 3D](https://en.wikipedia.org/wiki/3D_computer_graphics "3D computer graphics") graphics boards in arcade machines were introduced with the [Namco System 21](https://en.wikipedia.org/wiki/Namco_System_21 "Namco System 21")[\[47\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-47) and [Taito](https://en.wikipedia.org/wiki/Taito "Taito") Air System.[\[48\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-48)

### 1990s

[![](https://upload.wikimedia.org/wikipedia/commons/thumb/f/f8/DIAMONDSTEALTH3D2000-top.JPG/500px-DIAMONDSTEALTH3D2000-top.JPG)](https://en.wikipedia.org/wiki/File:DIAMONDSTEALTH3D2000-top.JPG) [S3 Graphics](https://en.wikipedia.org/wiki/S3_Graphics "S3 Graphics") [ViRGE](https://en.wikipedia.org/wiki/S3_ViRGE "S3 ViRGE")[![](https://upload.wikimedia.org/wikipedia/commons/thumb/8/88/Voodoo3-2000AGP.jpg/500px-Voodoo3-2000AGP.jpg)](https://en.wikipedia.org/wiki/File:Voodoo3-2000AGP.jpg) [Voodoo3](https://en.wikipedia.org/wiki/Voodoo3 "Voodoo3") 2000 AGP card

The 1990s again saw considerable advancements in professional workstation 3D graphics hardware from Sun Microsystems, SGI, and others. The introduction of [OpenGL](https://en.wikipedia.org/wiki/OpenGL "OpenGL") by SGI in 1992 paved the way for standard hardware-independent 3D programming interfaces.[\[49\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-49)[\[50\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Techspot-50) However, by the mid and late 90s, professional hardware was being slowly eclipsed by consumer products which offered similar or even better performance, especially in regards to texture mapping, at a lower cost and on platforms familiar to end users.[\[50\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Techspot-50)[\[51\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-Quantum-51)

In 1991, [S3 Graphics](https://en.wikipedia.org/wiki/S3_Graphics "S3 Graphics") introduced the _[S3 86C911](https://en.wikipedia.org/wiki/S3_Graphics "S3 Graphics")_, which its designers named after the [Porsche 911](https://en.wikipedia.org/wiki/Porsche_911 "Porsche 911") as an indication of the performance increase it promised.[\[52\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-52) The 86C911 spawned a variety of imitators: by 1995, all major PC graphics chip makers had added [2D](https://en.wikipedia.org/wiki/2D_computer_graphics "2D computer graphics") acceleration support to their chips.[\[53\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-53) Fixed-function _Windows accelerators_ surpassed expensive general-purpose graphics coprocessors in Windows performance, and such coprocessors faded from the PC market.

In the early- and mid-1990s, [real-time](https://en.wikipedia.org/wiki/Real-time_computer_graphics "Real-time computer graphics") 3D graphics became increasingly common in arcade, computer, and console games, which led to increasing public demand for hardware-accelerated 3D graphics. Early examples of mass-market 3D graphics hardware can be found in arcade system boards such as the [Sega Model 1](https://en.wikipedia.org/wiki/Sega_Model_1 "Sega Model 1"), [Namco System 22](https://en.wikipedia.org/wiki/Namco_System_22 "Namco System 22"), and [Sega Model 2](https://en.wikipedia.org/wiki/Sega_Model_2 "Sega Model 2"), and the [fifth-generation video game consoles](https://en.wikipedia.org/wiki/History_of_video_game_consoles_(fifth_generation) "History of video game consoles (fifth generation)") such as the [Saturn](https://en.wikipedia.org/wiki/Sega_Saturn "Sega Saturn"), [PlayStation](https://en.wikipedia.org/wiki/PlayStation_(console) "PlayStation (console)"), and [Nintendo 64](https://en.wikipedia.org/wiki/Nintendo_64 "Nintendo 64"). Arcade systems such as the Sega Model 2 and [SGI](https://en.wikipedia.org/wiki/Silicon_Graphics "Silicon Graphics") [Onyx](https://en.wikipedia.org/wiki/SGI_Onyx "SGI Onyx")-based Namco Magic Edge Hornet Simulator in 1993 were capable of hardware T&L ( [transform, clipping, and lighting](https://en.wikipedia.org/wiki/Transform,_clipping,_and_lighting "Transform, clipping, and lighting")) years before appearing in consumer graphics cards. [\[54\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-54)[\[55\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-55) In 1994, Sony used the term GPU (with the meaning graphics processing unit) in reference to the PlayStation console's Toshiba-designed Sony GPU.[\[3\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-:3-3)

Another early example is the [Super FX](https://en.wikipedia.org/wiki/Super_FX "Super FX") chip, a [RISC](https://en.wikipedia.org/wiki/Reduced_instruction_set_computer "Reduced instruction set computer")-based [on-cartridge graphics chip](https://en.wikipedia.org/wiki/ROM_cartridge#Use_in_hardware_enhancements "ROM cartridge") used in some [SNES](https://en.wikipedia.org/wiki/Super_Nintendo_Entertainment_System "Super Nintendo Entertainment System") games, notably _[Doom](https://en.wikipedia.org/wiki/List_of_Doom_ports#Super_NES "List of Doom ports")_ and _[Star Fox](https://en.wikipedia.org/wiki/Star_Fox_(1993_video_game) "Star Fox (1993 video game)")_. Some systems used [DSPs](https://en.wikipedia.org/wiki/Digital_signal_processor "Digital signal processor") to accelerate transformations. [Fujitsu](https://en.wikipedia.org/wiki/Fujitsu "Fujitsu"), which worked on the Sega Model 2 arcade system,[\[56\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-56) began working on integrating T&L into a single [LSI](https://en.wikipedia.org/wiki/Integrated_circuit "Integrated circuit") solution for use in home computers in 1995;[\[57\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-57) the Fujitsu Pinolite, the first 3D geometry processor for personal computers, announced in 1997.[\[58\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-58) The first hardware T&L GPU on [home](https://en.wikipedia.org/wiki/Home_console "Home console") [video game consoles](https://en.wikipedia.org/wiki/Video_game_console "Video game console") was the [Nintendo 64](https://en.wikipedia.org/wiki/Nintendo_64 "Nintendo 64")'s [Reality Coprocessor](https://en.wikipedia.org/wiki/Reality_Coprocessor "Reality Coprocessor"), released in 1996.[\[59\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-59) In 1997, [Mitsubishi](https://en.wikipedia.org/wiki/Mitsubishi "Mitsubishi") released the [3Dpro/2MP](https://en.wikipedia.org/wiki/AMD_FirePro "AMD FirePro"), a GPU capable of transformation and lighting, for [workstations](https://en.wikipedia.org/wiki/Workstation "Workstation") and [Windows NT](https://en.wikipedia.org/wiki/Windows_NT "Windows NT") desktops;[\[60\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-60) [ATi](https://en.wikipedia.org/wiki/ATi "ATi") used it for its [FireGL 4000](https://en.wikipedia.org/wiki/FireGL "FireGL") [graphics card](https://en.wikipedia.org/wiki/Graphics_card "Graphics card"), released in 1997.[\[61\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-61)

The term "GPU" was coined by [Sony](https://en.wikipedia.org/wiki/Sony "Sony") in reference to the 32-bit [Sony GPU](https://en.wikipedia.org/wiki/PlayStation_technical_specifications "PlayStation technical specifications") (designed by [Toshiba](https://en.wikipedia.org/wiki/Toshiba "Toshiba")) in the [PlayStation](https://en.wikipedia.org/wiki/PlayStation_(console) "PlayStation (console)") video game console, released in 1994.[\[62\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-gpu-62)

### 2000s

In October 2002, with the introduction of the [ATI](https://en.wikipedia.org/wiki/ATI_Technologies "ATI Technologies") _[Radeon 9700](https://en.wikipedia.org/wiki/Radeon_9700_core "Radeon 9700 core")_ (also known as R300), the world's first [Direct3D](https://en.wikipedia.org/wiki/Direct3D "Direct3D") 9.0 accelerator, pixel and vertex [shaders](https://en.wikipedia.org/wiki/Shader "Shader") could implement [looping](https://en.wikipedia.org/wiki/Loop_(computing) "Loop (computing)") and lengthy [floating point](https://en.wikipedia.org/wiki/Floating_point "Floating point") math, and were quickly becoming as flexible as CPUs, yet orders of magnitude faster for image-array operations. Pixel shading is often used for [bump mapping](https://en.wikipedia.org/wiki/Bump_mapping "Bump mapping"), which adds texture to make an object look shiny, dull, rough, or even round or extruded.[\[63\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-63)

With the introduction of the Nvidia [GeForce 8 series](https://en.wikipedia.org/wiki/GeForce_8_series "GeForce 8 series") and new generic stream processing units, GPUs became more generalized computing devices. [Parallel](https://en.wikipedia.org/wiki/Parallel_computing "Parallel computing") GPUs are making computational inroads against the CPU, and a subfield of research, dubbed GPU computing or [GPGPU](https://en.wikipedia.org/wiki/GPGPU "GPGPU") for _general purpose computing on GPU_, has found applications in fields as diverse as [machine learning](https://en.wikipedia.org/wiki/Machine_learning "Machine learning"),[\[64\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-64) [oil exploration](https://en.wikipedia.org/wiki/Oil_exploration "Oil exploration"), scientific [image processing](https://en.wikipedia.org/wiki/Image_processing "Image processing"), [linear algebra](https://en.wikipedia.org/wiki/Linear_algebra "Linear algebra"),[\[65\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-65) [statistics](https://en.wikipedia.org/wiki/Statistics "Statistics"),[\[66\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-66) [3D reconstruction](https://en.wikipedia.org/wiki/3D_reconstruction "3D reconstruction"), and [stock options](https://en.wikipedia.org/wiki/Stock_options "Stock options") pricing. GPGPUs were the precursors to what is now called a compute shader (e.g. [CUDA](https://en.wikipedia.org/wiki/CUDA "CUDA"), [OpenCL](https://en.wikipedia.org/wiki/OpenCL "OpenCL"), [DirectCompute](https://en.wikipedia.org/wiki/DirectCompute "DirectCompute")) and actually abused the hardware to a degree by treating the data passed to algorithms as texture maps and executing algorithms by drawing a triangle or quad with an appropriate pixel shader.\[ _[clarification needed](https://en.wikipedia.org/wiki/Wikipedia:Please_clarify "Wikipedia:Please clarify")_\] This entails some overheads since units like the [scan converter](https://en.wikipedia.org/wiki/Rasterization "Rasterization") are involved where they are not needed (nor are triangle manipulations even a concern—except to invoke the pixel shader).\[ _[clarification needed](https://en.wikipedia.org/wiki/Wikipedia:Please_clarify "Wikipedia:Please clarify")_\]

Nvidia's CUDA platform, first introduced in 2007,[\[67\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-67) was the earliest widely adopted programming model for GPU computing. OpenCL is an open standard defined by the [Khronos Group](https://en.wikipedia.org/wiki/Khronos_Group "Khronos Group") that allows for the development of code for both GPUs and CPUs with an emphasis on portability.[\[68\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-68) OpenCL solutions are supported by Intel, AMD, Nvidia, and ARM, and according to a report in 2011 by [Evans Data](https://en.wikipedia.org/wiki/Evans_Data_Corporation "Evans Data Corporation"), OpenCL had become the second most popular HPC tool.[\[69\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-69)

### 2010s

In 2010, Nvidia partnered with [Audi](https://en.wikipedia.org/wiki/Audi "Audi") to power their cars' dashboards, using the [Tegra](https://en.wikipedia.org/wiki/Tegra "Tegra") GPU to provide increased functionality to cars' navigation and entertainment systems.[\[70\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-70) Advances in GPU technology in cars helped advance [self-driving technology](https://en.wikipedia.org/wiki/Autonomous_car "Autonomous car").[\[71\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-71) AMD's [Radeon HD 6000 series](https://en.wikipedia.org/wiki/Radeon_HD_6000_series "Radeon HD 6000 series") cards were released in 2010, and in 2011 AMD released its 6000M Series discrete GPUs for mobile devices.[\[72\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-72) The [Kepler line](https://en.wikipedia.org/wiki/Kepler_(microarchitecture) "Kepler (microarchitecture)") of graphics cards by Nvidia were released in 2012 and were used in the Nvidia 600 and 700 series cards. A feature in this GPU microarchitecture included GPU boost, a technology that adjusts the clock-speed of a video card to increase or decrease according to its power draw.[\[73\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-73) Kepler also introduced [NVENC](https://en.wikipedia.org/wiki/NVENC "NVENC") video encoding acceleration technology.

The [PS4](https://en.wikipedia.org/wiki/PlayStation_4_technical_specifications "PlayStation 4 technical specifications") and [Xbox One](https://en.wikipedia.org/wiki/Xbox_One "Xbox One") were released in 2013; they both used GPUs based on [AMD's Radeon HD 7850 and 7790](https://en.wikipedia.org/wiki/Radeon_HD_7000_series "Radeon HD 7000 series").[\[74\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-74) Nvidia's Kepler line of GPUs was followed by the [Maxwell](https://en.wikipedia.org/wiki/Maxwell_(microarchitecture) "Maxwell (microarchitecture)") line, manufactured on the same process. Nvidia's 28 nm chips were manufactured by [TSMC](https://en.wikipedia.org/wiki/TSMC "TSMC") in Taiwan using the 28 nm process. Compared to the 40 nm technology from the past, this manufacturing process allowed a 20 percent boost in performance while drawing less power.[\[75\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-75)[\[76\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-76) [Virtual reality headsets](https://en.wikipedia.org/wiki/Virtual_reality_headset "Virtual reality headset") have high system requirements; manufacturers recommended the GTX 970 and the R9 290X or better at the time of their release.[\[77\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-77)[\[78\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-78) Cards based on the [Pascal](https://en.wikipedia.org/wiki/Pascal_(microarchitecture) "Pascal (microarchitecture)") microarchitecture were released in 2016. The [GeForce 10 series](https://en.wikipedia.org/wiki/GeForce_10_series "GeForce 10 series") of cards are of this generation of graphics cards. They are made using the 16nm manufacturing process which improves upon previous microarchitectures.[\[79\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-79)

In 2018, Nvidia launched the RTX 20 series GPUs that added [ray tracing](https://en.wikipedia.org/wiki/Ray_tracing_(graphics) "Ray tracing (graphics)") cores to GPUs, allowing real time ray tracing to be performant on mass market hardware.[\[80\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-:0-80) [Polaris 11](https://en.wikipedia.org/wiki/Polaris_11 "Polaris 11") and [Polaris 10](https://en.wikipedia.org/wiki/Polaris_10 "Polaris 10") GPUs from AMD are fabricated by a 14 nm process. Their release resulted in a substantial increase in the performance per watt of AMD video cards.[\[81\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-81) AMD also released the Vega GPU series for the high end market as a competitor to Nvidia's high end Pascal cards, also featuring [HBM2](https://en.wikipedia.org/wiki/High_Bandwidth_Memory "High Bandwidth Memory") like the Titan V.\[ _[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_\]

In 2019, AMD released the successor to their [Graphics Core Next](https://en.wikipedia.org/wiki/Graphics_Core_Next "Graphics Core Next") (GCN) microarchitecture/instruction set. Dubbed [RDNA](https://en.wikipedia.org/wiki/RDNA_(microarchitecture) "RDNA (microarchitecture)"), the first product featuring it was the [Radeon RX 5000 series](https://en.wikipedia.org/wiki/Radeon_RX_5000_series "Radeon RX 5000 series") of video cards.[\[82\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-amd-official-press-release-computex-82) The company announced that the successor to the RDNA microarchitecture would be incremental (a "refresh"). AMD unveiled the [Radeon RX 6000 series](https://en.wikipedia.org/wiki/Radeon_RX_6000_series "Radeon RX 6000 series"), its [RDNA 2](https://en.wikipedia.org/wiki/RDNA_2 "RDNA 2") graphics cards with support for hardware-accelerated ray tracing.[\[83\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-83) The product series, launched in late 2020, consisted of the RX 6800, RX 6800 XT, and RX 6900 XT.[\[84\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-84)[\[85\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-RDNA2InfinityCache-85) The RX 6700 XT, which is based on Navi 22, was launched in early 2021.[\[86\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-86)

The [PlayStation 5](https://en.wikipedia.org/wiki/PlayStation_5 "PlayStation 5") and [Xbox Series X and Series S](https://en.wikipedia.org/wiki/Xbox_Series_X_and_Series_S "Xbox Series X and Series S") were released in 2020; they both use GPUs based on the RDNA 2 microarchitecture with incremental improvements and different GPU configurations in each system's implementation.[\[87\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-RDNA2PS5Console-87)[\[88\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-88)[\[89\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-RDNA2XboxConsoles-89)

### 2020s

See also: [AI accelerator](https://en.wikipedia.org/wiki/AI_accelerator "AI accelerator")

In the 2020s, GPUs have been increasingly used for calculations involving [embarrassingly parallel](https://en.wikipedia.org/wiki/Embarrassingly_parallel "Embarrassingly parallel") problems, such as training of [neural networks](https://en.wikipedia.org/wiki/Artificial_neural_network#Training "Artificial neural network") on enormous datasets that are needed for artificial intelligence [large language models](https://en.wikipedia.org/wiki/Large_language_model "Large language model"). Specialized processing cores on most modern GPUs that are dedicated to [deep learning](https://en.wikipedia.org/wiki/Deep_learning "Deep learning") provide significant [FLOPS](https://en.wikipedia.org/wiki/Floating_point_operations_per_second "Floating point operations per second") performance increases, using 4×4 matrix multiplication and division. Early implementations, such as Nvidia's [Volta](https://en.wikipedia.org/wiki/Volta_(microarchitecture) "Volta (microarchitecture)") microarchitecture, released in 2017,[\[90\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-90) saw results of up to 128 TFLOPS in some applications.[\[91\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-91)

Since then, AI acceleration cores have been a widely adopted feature in consumer and workstation microarchitectures starting with Nvidia's [Turing](https://en.wikipedia.org/wiki/Turing_(microarchitecture) "Turing (microarchitecture)") microarchitecture in 2018,[\[80\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-:0-80) named Tensor cores. Originally used for [Deep Learning Super Sampling](https://en.wikipedia.org/wiki/Deep_Learning_Super_Sampling "Deep Learning Super Sampling") (DLSS) to enhance gaming performance and improve image quality, they have since been used in Nvidia's [Broadcast](https://www.nvidia.com/en-us/geforce/broadcasting/broadcast-app/) software to provide many AI powered effects such as voice filtering and video noise removal and in other software such as [Blender](https://en.wikipedia.org/wiki/Blender_(software) "Blender (software)") for DLSS in the view port.

AMD originally implemented their equivalent "Matrix" Cores for consumers in their [RDNA 3](https://en.wikipedia.org/wiki/RDNA_3 "RDNA 3") architecture - however RDNA 4's Matrix Cores were the first to introduce FP8 acceleration - which is required to run the full [FSR Redstone](https://www.amd.com/en/products/graphics/technologies/fidelityfx/super-resolution.html) feature set such as Machine Learning Upscaling and Frame Generation. However, community made hacks on both Linux and Windows have allowed RDNA 2, 3, and competitor GPUs to run a weaker version of FSR 4 known as FSR 4 INT8. The [PlayStation 5 Pro](https://en.wikipedia.org/wiki/PlayStation_5_Pro "PlayStation 5 Pro"), launched in 2024, has customized Machine Learning cores - which focus solely on INT8 acceleration - based on RDNA 4's for PlayStation Spectral Super Resolution to enhance framerates and image quality.

Intel has implemented their equivalent "XMX" Cores in all of their [Arc](https://en.wikipedia.org/wiki/Intel_Arc "Intel Arc") GPUs, starting with the [Alchemist](https://en.wikipedia.org/wiki/Intel_Arc#Alchemist "Intel Arc") microarchitecture. This is used for XeSS (Xe Super Sampling), XeFG (Xe Frame Generation), and more.

Ray tracing has also been incredibly prevalent in the 2020s with some games, such as [DOOM: The Dark Ages](https://en.wikipedia.org/wiki/Doom:_The_Dark_Ages "Doom: The Dark Ages"), requiring a hardware ray tracing capable GPU to even start[\[92\]](https://en.wikipedia.org/wiki/Graphics_processing_unit#cite_note-92). While it does lead to worse performance and less overall accessibility, [id software](https://en.wikipedia.org/wiki/Id_Software "Id Software") claimed this saved several man hours and shrinked the game by over 100 gigabytes due to the game being entirely built around ray tracing \[ _[citation needed](https://en.wikipedia.org/wiki/Wikipedia:Citation_needed "Wikipedia:Citation needed")_\].
